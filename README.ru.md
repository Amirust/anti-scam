# Mr Beast / Melstroy anti-scam Discord bot

🇬🇧 [English version](README.md)

Discord-бот, который ловит известные скам-картинки (фейковые розыгрыши,
казино-спам и подобное), рассылаемые со взломанных аккаунтов, банит
отправителя и отчитывается в админ-канал.

Схема скамеров всегда одна: угнали аккаунт — и один и тот же скриншот летит в
десятки каналов. Сравнение по байтам не работает — каждая копия пережата,
отресайжена или слегка подредактирована, поэтому бот использует перцептивные
хеши, откалиброванные так, чтобы переживать именно эти трансформации.

## Что делает бот

Когда приходит сообщение с картинкой во вложении, бот:

1. Скачивает картинку (одинаковые картинки от одного автора, обрабатываемые
   одновременно дедуплицируются на лету).
2. Прогоняет её через пайплайн детекта (ниже) по датасету известных
   скам-картинок.
3. Исполняет вердикт:
   - **Ban** — отправляет пользователю объяснение в личку (best-effort), банит
     его с именем совпавшей записи датасета в причине для аудит-лога, удаляет
     его недавние сообщения и постит отчёт-эмбед с картинкой в админ-канал
     гильдии. Если у бота не хватает прав на бан — вместо этого постит отчёт
     «не смог забанить».
   - **Review** — совпадение слабее порога бана; постит отчёт с картинкой в
     админ-канал с кнопкой **Ban user** (для модераторов с правом Ban Members;
     заодно удаляет сообщения пользователя начиная с помеченного поста, максимум
     за 7 дней — лимит Discord). Состояние кнопок живёт в самом сообщении,
     поэтому они переживают рестарты бота.
   - **Clean** — ничего не происходит.

На каждом отчёте есть также кнопка **Add to dataset** (только для владельца
бота): она хеширует приложенную картинку, дописывает её в локальный
`banned.json` и перезагружает датасет на лету — рестарт не нужен. Картинки,
уже жёстко совпадающие с существующей записью, отклоняются как дубликаты.

## Как работает детект

Сырые хеши (sha256) здесь бесполезны: скамеры пережимают каждую копию, байты
всегда разные. Поэтому:

**0. Нормализация.** Каждая картинка ресайзится до 256×256 (Lanczos) и
переводится в оттенки серого.

**1. Перцептивный хеш целой картинки.** pHash 8×8 (DCT median) всей картинки
сравнивается с каждой записью датасета. Дистанция Хэмминга ≤ 10 из 64 бит —
жёсткое совпадение. Откалибровано на эталонном наборе: пережатые копии одной
картинки дают 0–6, чужие пары — 18+.

**2. Тайловая проверка.** Если целая картинка не совпала, бот ищет частично
перерисованные варианты. Картинка режется на сетку 4×4 из тайлов 64×64,
каждый хешируется отдельно. Тайловый pHash разваливается уже от сдвига в 2 px,
а пережатые копии гуляют на ~5 px, поэтому входящая картинка сначала
примеряется на всех сдвигах в пределах ±6 px с шагом 2 px (49 вариантов), и
берётся сетка с лучшим выравниванием.

**3. Информативность тайла.** Тайл учитывается, только если дисперсия его
яркости больше 150 — плоские фоны дают меньше ~120, реальный контент — от
~200. Это не даёт пустым полям голосовать.

**4. Подсчёт.** Для каждой записи датасета тайл считается *информативным*,
если он информативен в обеих картинках, и *совпавшим*, если дистанция Хэмминга
≤ 13. Если сравнили больше 6 информативных тайлов:

- совпало ≥ 75% информативных → **Ban**
- совпало ≥ 60% → **Review**
- иначе → **Clean**

Все пороги настраиваются через [`config.toml`](#конфигурация).

## Датасет запрещённых картинок

На старте бот читает `banned.json` (путь меняется через env-переменную
`BANNED_CONFIG`). Нет файла — бот пишет предупреждение в лог и стартует с
пустым датасетом. В файле только хеши — исходные картинки из него
восстановить нельзя.

Взять датасет можно двумя способами:

- **Скачать официальный:** `./get-config.sh` качает JSON из releases
  репозитория и сверяет sha256 с `banned.json.sha256`. Если локальный
  `banned.json` уже существует, скрипт спросит перед перезаписью (локальные
  добавления иначе пропадут). Перед запуском скрипт стоит прочитать.
- **Собрать свой:** сложите скриншоты скама в папку и запустите
  `anti-scam export <папка> [banned.json]`. Каждая картинка прогоняется через
  хеш-пайплайн; команда пишет конфиг и печатает его sha256 для файла
  контрольной суммы. На один тип скама обычно хватает одного-двух эталонов.

Датасет привязан к версии хеш-пайплайна (`pipeline_version`). Если алгоритм
хеширования поменялся, бот не примет старый конфиг — скачайте свежий или
перегенерируйте через `anti-scam export`.

## Установка

Требования: Rust (edition 2024).

```sh
export DISCORD_TOKEN=токен-бота
cargo run --release
```

База sqlite (`data.db`) создаётся и мигрируется автоматически при первом
запуске.

Боту нужен gateway-интент **Message Content** (включается в Discord developer
portal) и права **Ban Members**, **Send Messages** и **Embed Links** в
гильдии.

Дальше в каждой гильдии администратор задаёт канал для отчётов:

```
/settings set_notification_channel #канал
```

Настройки гильдий хранятся в sqlite и отдаются из LRU-кэша в памяти, поэтому
обычная обработка сообщений базу не трогает.

### Docker

```sh
cp .env.example .env          # впишите туда DISCORD_TOKEN
mkdir -p config data
cp config.toml config/        # опционально, без него работают дефолты
cp banned.json config/        # или скачайте: ./get-config.sh config/banned.json
docker compose up -d --build
```

`banned.json` и `config.toml` лежат на хосте в `./config` и монтируются в
контейнер директорией — записи, добавленные кнопкой **Add to dataset**,
попадают в файл на хосте. База sqlite сохраняется в `./data`.

## Конфигурация

Опциональна. Бот читает `config.toml` из рабочей директории (путь меняется
через env-переменную `CONFIG_PATH`); отсутствующие ключи или отсутствующий
файл откатываются на дефолты. Файл читается один раз на старте.

| Ключ | Дефолт | Значение |
|------|--------|----------|
| `detection.whole_match_threshold` | 10 | Макс. дистанция Хэмминга (из 64 бит) для совпадения целой картинки |
| `detection.tile_match_threshold` | 13 | Макс. дистанция Хэмминга для совпадения тайла |
| `detection.min_informative_tiles` | 6 | Минимум информативных тайлов для доверия тайловому вердикту |
| `detection.hard_match_percent` | 75 | Процент совпавших тайлов для автобана |
| `detection.review_percent` | 60 | Процент совпавших тайлов для отправки на ручную проверку |
| `cache.guild_settings_capacity` | 100 | Сколько гильдий держится в LRU-кэше настроек |

Это пороги времени сравнения — их подкрутка никогда не инвалидирует
существующий `banned.json`.

### Переменные окружения

| Переменная | Дефолт | Значение |
|------------|--------|----------|
| `DISCORD_TOKEN` | — (обязательна) | Токен бота |
| `BANNED_CONFIG` | `./banned.json` | Путь к датасету запрещённых картинок |
| `CONFIG_PATH` | `./config.toml` | Путь к рантайм-конфигу |
